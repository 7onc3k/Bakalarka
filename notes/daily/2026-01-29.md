OK, tady je můj návrh — tři skupiny cílového chování + metriky.

  Cílové chování agenta

  A. Porozumění a plánování

  #: B1
  Chování: Přečte specifikaci před kódem
  Proč: Bez toho implementuje z weights, ne z požadavků
  Jak ověřit: Transcript: první tool call = čtení Issue #1
  ────────────────────────────────────────
  #: B2
  Chování: Decomponuje do sub-issues
  Proč: Menší úkoly = vyšší úspěšnost (Ehsani 2026)
  Jak ověřit: gh issue list — vznikly issues?
  ────────────────────────────────────────
  #: B3
  Chování: Vytvoří architecture issue před implementací
  Proč: Plánování struktury před kódováním
  Jak ověřit: GitHub: issue s architekturou existuje před prvním kódem

  B. Vývojový proces

  #: B4
  Chování: TDD test-first ze spec
  Proč: Obrana proti test oracle problému (Mathews 2024)
  Jak ověřit: Git log: test soubor před src souborem per branch
  ────────────────────────────────────────
  #: B5
  Chování: Branch per issue + PR workflow
  Proč: Izolace změn, traceability
  Jak ověřit: git branch -a, gh pr list
  ────────────────────────────────────────
  #: B6
  Chování: Průběžné commity
  Proč: Umožňuje retrospektivní analýzu procesu
  Jak ověřit: Commit count > 1 per branch
  ────────────────────────────────────────
  #: B7
  Chování: Lint + typecheck čistý
  Proč: Základní kvalita kódu, agent opravuje problémy
  Jak ověřit: npm run lint && tsc --noEmit — agent spustí a fixne

  C. Kvalita výstupu

  #: Q1
  Metrika: Ref test pass rate
  Co měří: Funkční korektnost
  Jak: 40 ref testů proti agentovu kódu
  Typ: Automatická
  ────────────────────────────────────────
  #: Q2
  Metrika: Mutation score
  Co měří: Kvalita agentových testů
  Jak: Stryker
  Typ: Automatická
  ────────────────────────────────────────
  #: Q3
  Metrika: AC coverage
  Co měří: Úplnost implementace
  Jak: Kolik z 24 AC pokryto testy
  Typ: Manuální/LLM
  ────────────────────────────────────────
  #: Q4
  Metrika: Lint warnings + typecheck errors
  Co měří: Statická kvalita
  Jak: ESLint + tsc
  Typ: Automatická
  ────────────────────────────────────────
  #: Q5
  Metrika: Strict mode compliance
  Co měří: Typová bezpečnost
  Jak: Počet any, strict flags
  Typ: Automatická
  ────────────────────────────────────────
  #: Q6
  Metrika: Složitost kódu
  Co měří: Udržovatelnost
  Jak: Cyklomatická složitost (ESLint), max function length
  Typ: Automatická
  ────────────────────────────────────────
  #: Q7
  Metrika: Kvalita kódu (subjektivní)
  Co měří: Čitelnost, patterny, idiomatičnost
  Jak: LLM-as-judge s rubrikem
  Typ: LLM
  ────────────────────────────────────────
  #: Q8
  Metrika: Kvalita commit messages a issues
  Co měří: Procesní dokumentace
  Jak: LLM-as-judge s rubrikem
  Typ: LLM

  D. Efektivita
